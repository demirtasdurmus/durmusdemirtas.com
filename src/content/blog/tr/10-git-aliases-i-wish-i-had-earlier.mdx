Git, modern versiyon kontrol sistemlerinin temel taşlarından biridir ve geliştiricilerin projelerini kolay bir şekilde yönetebilmelerini sağlar. Ancak en deneyimli Git kullanıcıları için bile, sık sık tekrarlanan bazı işleri otomatik hale getirmek faydalı olabilir.

Git alias'ları, Git komutlarını kendi çalışma şeklinize göre özelleştirmenin bir yoludur. Bu yazıda sizlerle geliştirme sürecimi daha akıcı ve hızlı hale getiren en etkili Git alias'larımı paylaşacağım. Bu alias'lar commit'leri geri alma, branch'ler arasında gezinme veya repository yazarlarını yönetme gibi birçok konuda size zaman kazandırır ve hata yapma riskini azaltır.

Öncelikle tüm listeyi paylaşacak ve bunları Git konfigurasyon dosyanıza nasıl ekleyebileceğinizi göstereceğim. Ardından her birini tek tek ele alarak kullanım yerlerini ve sağladıkları avantajları açıklayacağım. Bu alias'lar macOS işletim sistemi için hazırlanmıştır ancak diğer işletim sistemlerine de kolayca uyarlanabilir.

Hadi başlayalım.

Bir terminal açarak `.gitconfig` dosyanızı tercih ettiğiniz bir kod editöründe görüntüleyin. Ben VS Code kullanacağım:

```bash
code ~/.gitconfig
```

Aşağıdaki alias'ları mevcut alias bölümünüzün altına yapıştırın. Eğer açılan dosyada henüz alias'lar için bir bölüm yoksa ekleyin.

```bash
# ...diğer konfigürasyonlar
[alias]
  # Son commiti geri alma
  u = reset --soft HEAD~1
  # Commit geçmişini kısa ve okunaklı format ile gösterir
  # commitHash - (tag-HEAD) commitMesajı (şimdiye kadar geçen zaman) - [commitAtanKişininAdı]
  ls = log --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %C(bold)%s%Creset %Cgreen(%cr) %C(bold blue)[%an]%Creset' --abbrev-commit --decorate
  # Commit geçmişini detaylı olarak gösterir
  # commitHash (tag-HEAD) commitMesajı commitAtanKişininAdı
  # eklenenDosyaSayısı silinenDosyaSayısı
  ll = log --pretty=format:"%C(yellow)%h%Cred%d\\ %Creset%s%Cblue\\ [%an]" --decorate --numstat
  # Conflict'leri görüntüler
  cf = diff --name-only --diff-filter=U
  # Lokal branch'leri liste formatında görüntüler
  lb = !git branch -vv | cut -c 3- | awk '$3 { print $1 }'
  # Tüm repository yazarlarını liste formatında görüntüler
  au = "!sh -c \"git log --format='%aN - [%aE]' | grep -v 'users.noreply.github.com' | sort -u --ignore-case | awk -F' - ' '{printf \\\"\\033[33m%s\\033[0m - \\033[34m%s\\033[0m\\\\n\\\", \\$1, substr(\\$2, 2, length(\\$2)-2)}'\""
  # fzf ile lokal branch'leri görüntüle ve bir tanesini seçerek branch değiştir
  clb = "!sh -c 'git branch | grep -v \"^\\*\" | fzf --height=20% --reverse --info=inline | xargs git checkout'"
  # fzf ile remote branch'leri görüntüle ve bir tanesini seçerek branch değiştir
  crb = "!sh -c 'git branch -r | grep -v \"^\\*\" | sed \"s|origin/||\" | fzf --height=20% --reverse --info=inline | xargs git checkout'"
  # fzf ile birden fazla dosya seçerek staging'e atmaya yarar
  afz = "!sh -c 'git ls-files -m -o --exclude-standard | fzf -m --print0 | xargs -0 git add'"
  # Tüm alias'ları görüntüle
  la = "!sh -c 'git config -l | grep alias | cut -c 7-'"
# ...diğer konfigürasyon seçenekleri
```

Dosyayı kaydedin ve mevcut git repository’lerinizden birine gidin. Şimdi bu alias’ları tek tek inceleyelim ve kullanım alanlarını görelim.

### 1. Son Commit'i Geri Alma

```bash
# Alias
u = reset --soft HEAD~1

# Kullanımı
git u
```

Son commit’i hızlıca geri alır ve değişiklikleri working directory’de tutar. Son commit’te bir hata fark ettiğinizde ve düzeltme yapmanız gerektiğinde oldukça kullanışlıdır.

### 2. Kısa Commit Geçmişi

```bash
# Alias
ls = log --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %C(bold)%s%Creset %Cgreen(%cr) %C(bold blue)[%an]%Creset' --abbrev-commit --decorate

# Kullanımı
git ls
```

Commit geçmişini kısa ve görsel olarak anlaşılır bir formatta gösterir. Commit hash’i, tagler, mesaj, zaman ve author bilgilerini içerir. Repository geçmişine hızlıca göz atmak için idealdir.

### 3. Detaylı Commit Geçmişi

```bash
# Alias
ll = log --pretty=format:"%C(yellow)%h%Cred%d\\ %Creset%s%Cblue\\ [%an]" --decorate --numstat

# Kullanımı
git ll
```

Commit hash’i, tagler, mesaj, author ve dosya bazlı değişiklikleri (eklenen ve silinen satırlar) içeren detaylı bir log sunar. Özellikle code review veya debugging sırasında oldukça faydalıdır.

### 4. Conflict'leri Görüntüleme

```bash
# Alias
cf = diff --name-only --diff-filter=U

# Kullanımı
git cf
```

Merge sırasında oluşan conflict'leri listeler. Bunları hızlıca tespit edip çözmenizi sağlar.

### 5. Lokal Branch'leri Görüntüleme

```bash
# Alias
lb = !git branch -vv | cut -c 3- | awk '$3 { print $1 }'

# Kullanımı
git lb
```

Lokal branch'leri sade ve okunabilir bir formatta listeler. Branch yönetimini kolaylaştırır.

### 6. Repository Yazarlarını Görüntüleme

```bash
# Alias
au = "!sh -c \"git log --format='%aN - [%aE]' | grep -v 'users.noreply.github.com' | sort -u --ignore-case | awk -F' - ' '{printf \\\"\\033[33m%s\\033[0m - \\033[34m%s\\033[0m\\\\n\\\", \\$1, substr(\\$2, 2, length(\\$2)-2)}'\""

# Kullanımı
git au
```

Repository yazarlarını, anonim GitHub kullanıcılarını hariç tutarak, sade bir formatta listeler. Projeye katkıda bulunanları görmek için idealdir.

> Bu yazım tarzının neden diğerlerinden biraz farklı olduğunu ve `!sh -c` ile başladığını merak ediyor olabilirsiniz. Bu, komut daha karmaşık shell işlemleri içerdiğinde (örneğin `|`), mantıksal işlemler (`&&`) veya döngüler veya koşullar gibi gelişmiş shell özellikleri gerektirdiğinde kullanılmalıdır.

### 7. fzf ile Lokal Branch'e Geçiş

```bash
# Alias
clb = "!sh -c 'git branch | grep -v \"^\\*\" | fzf --height=20% --reverse --info=inline | xargs git checkout'"

# Kullanımı
git clb
```

<Callout>
  Bu alias ve aşağıdaki iki alias'ı kullanabilmek için bilgisayarınızda
  [fzf](https://github.com/junegunn/fzf) kurulu olmalıdır. Kurulum seçenekleri için [bu
  bağlantıya](https://github.com/junegunn/fzf?tab=readme-ov-file#installation) göz atabilirsiniz.
</Callout>

`fzf` kullanarak local branch'leri listeler ve etkileşimli olarak checkout yapmanızı sağlar. Çok sayıda branch olan projelerde büyük kolaylık sağlar.

### 8. fzf ile Remote Branch'e Geçiş

```bash
# Alias
crb = "!sh -c 'git branch -r | grep -v \"^\\*\" | sed \"s|origin/||\" | fzf --height=20% --reverse --info=inline | xargs git checkout'"

# Kullanımı
git crb
```

Yukarıdaki alias ile aynı mantıkta çalışır; tek fark remote branch'leri listelemesidir.

### 9. fzf ile Birden fazla Dosyayı Staging’e Ekleme

```bash
# Alias
afz = "!sh -c 'git ls-files -m -o --exclude-standard | fzf -m --print0 | xargs -0 git add'"

# Kullanımı
git afz
```

`fzf` kullanarak dosyaları çoklu seçimle stage etmenizi sağlar. Ok tuşlarıyla gezinip, tab ile seçim yapabilir ve enter ile ekleyebilirsiniz.

### 10. Tüm Git Alias'larını Listeleme

```bash
# Alias
la = "!sh -c 'git config -l | grep alias | cut -c 7-'"

# Kullanımı
git la
```

Konfigürasyon dosyanıza eklenmiş tüm Git alias'larını listeler. Alias'lara alışana kadar oldukça kullanışlıdır.

### Sonuç

Git alias'ları, Git’in ne kadar esnek ve güçlü olduğunu gösteren harika bir örnektir. Çalışma şeklinize uygun düzenlemeler yaparak verimliliğinizi ciddi şekilde artırabilirsiniz. Commit geri alma, branch yönetimi, repository yazarlarını görüntüleme ve dosya stage etme gibi günlük işlemleri hızlandıran bu alias'lar, Git kullanım deneyiminizi bir üst seviyeye taşır. Siz de kendi akışınıza uygun başka alias'lar bulabilir ve Git deneyiminizi iyileştirebilirsiniz.
