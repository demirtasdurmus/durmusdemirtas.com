Git, modern versiyon kontrol sistemlerinin temel taşlarından biridir ve geliştiricilerin projelerini hassasiyetle yönetmesini sağlar. Ancak en deneyimli Git kullanıcıları için bile, tekrarlayan işleri otomatik hale getirmek faydalı olabilir.

Git aliasları, Git komutlarını kendi çalışma şeklinize göre özelleştirmenin bir yoludur. Bu yazıda sizlerle geliştirme sürecimi daha akıcı ve hızlı hale getiren en etkili Git aliaslarımı paylaşacağım. Bu aliaslar commitleri geri alma, branchler arasında gezinme veya repository yazarlarını yönetme gibi birçok konuda size zaman kazandırır ve hata riskini azaltır.

Öncelikle tüm listeyi paylaşacak ve bunları Git konfigurasyon dosyanıza nasıl ekleyebileceğinizi göstereceğim. Ardından her birini tek tek ele alarak kullanım senaryolarını ve sağladıkları avantajları açıklayacağım. Bu aliaslar macOS işletim sistemi için yapılmıştır ancak diğer işletim sistemlerine de kolayca uyarlanabilir.

Hadi başlayalım.

Bir terminal açın ve `.gitconfig` dosyanızı tercih ettiğiniz bir kod editörü ile açın. Ben VS Code kullanacağım:

```bash
code ~/.gitconfig
```

Eğer açılan dosyada henüz aliaslar için bir bölümü yoksa ekleyin ya da aşağıdaki aliasları mevcut alias bölümünüzün altına yapıştırın:

```bash
# ...other configuration options
[alias]
  # Son commiti geri alma
  u = reset --soft HEAD~1
  # Commit geçmişini kısa ve renkli bir format ile gösterir
  # commit hashı - (tag-HEAD) commit mesajı (şimdiye kadar geçen zaman) - [commit atan kişinin adı]
  ls = log --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %C(bold)%s%Creset %Cgreen(%cr) %C(bold blue)[%an]%Creset' --abbrev-commit --decorate
  # Commit geçmişini detaylı olarak gösterir
  # commit hashı (tag-HEAD) commit mesajı commit atan kişinin adı
  # eklenen dosya sayısı silinen dosya sayısı
  ll = log --pretty=format:"%C(yellow)%h%Cred%d\\ %Creset%s%Cblue\\ [%an]" --decorate --numstat
  # Conflictleri görüntüler
  cf = diff --name-only --diff-filter=U
  # Lokal branchleri liste formatında görüntüler
  lb = !git branch -vv | cut -c 3- | awk '$3 { print $1 }'
  # Tüm katkıda bulunanları liste formatında görüntüler
  au = "!sh -c \"git log --format='%aN - [%aE]' | grep -v 'users.noreply.github.com' | sort -u --ignore-case | awk -F' - ' '{printf \\\"\\033[33m%s\\033[0m - \\033[34m%s\\033[0m\\\\n\\\", \\$1, substr(\\$2, 2, length(\\$2)-2)}'\""
  # fzf ile lokal branchleri görüntüle ve bir tanesini seçerek branch değiştir
  clb = "!sh -c 'git branch | grep -v \"^\\*\" | fzf --height=20% --reverse --info=inline | xargs git checkout'"
  # fzf ile remote branchleri görüntüle ve bir tanesini seçerek branch değiştir
  crb = "!sh -c 'git branch -r | grep -v \"^\\*\" | sed \"s|origin/||\" | fzf --height=20% --reverse --info=inline | xargs git checkout'"
  # fzf ile birden fazla dosya seçerek staginge atmaya yarar
  afz = "!sh -c 'git ls-files -m -o --exclude-standard | fzf -m --print0 | xargs -0 git add'"
  # Tüm aliasları görüntüle
  la = "!sh -c 'git config -l | grep alias | cut -c 7-'"
# ...diğer konfigürasyon seçenekleri
```

Dosyayı kaydedin ve mevcut git repository’lerinizden birine gidin. Şimdi bu alias’ları tek tek inceleyelim ve kullanım alanlarını görelim.

### 1. Son Commiti Geri Alma

```bash
# Alias
u = reset --soft HEAD~1

# Kullanımı
git u
```

Son commit’i hızlıca geri alır ve değişiklikleri working directory’de tutar. Son commit’te bir hata fark ettiğinizde ve düzeltme yapmanız gerektiğinde oldukça kullanışlıdır.

### 2. Kısa Commit Geçmişi

```bash
# Alias
ls = log --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %C(bold)%s%Creset %Cgreen(%cr) %C(bold blue)[%an]%Creset' --abbrev-commit --decorate

# Kullanımı
git ls
```

Commit geçmişini kısa ve görsel olarak anlaşılır bir formatta gösterir. Commit hash’i, tagler, mesaj, zaman ve author bilgilerini içerir. Repository geçmişine hızlıca göz atmak için idealdir.

### 3. Detaylı Commit Geçmişi

```bash
# Alias
ll = log --pretty=format:"%C(yellow)%h%Cred%d\\ %Creset%s%Cblue\\ [%an]" --decorate --numstat

# Kullanımı
git ll
```

Commit hash’i, tagler, mesaj, author ve dosya bazlı değişiklikleri (eklenen ve silinen satırlar) içeren detaylı bir log sunar. Özellikle code review veya debugging sırasında oldukça faydalıdır.

### 4. Conflictleri Görüntüleme

```bash
# Alias
cf = diff --name-only --diff-filter=U

# Kullanımı
git cf
```

Merge sırasında oluşan conflictleri listeler. Bunları hızlıca tespit edip çözmenizi sağlar.

### 5. Lokal Branchleri Görüntüleme

```bash
# Alias
lb = !git branch -vv | cut -c 3- | awk '$3 { print $1 }'

# Kullanımı
git lb
```

Lokal branchleri sade ve okunabilir bir formatta listeler. Branch yönetimini kolaylaştırır.

### 6. Repository authorlarını Görüntüleme

```bash
# Alias
au = "!sh -c \"git log --format='%aN - [%aE]' | grep -v 'users.noreply.github.com' | sort -u --ignore-case | awk -F' - ' '{printf \\\"\\033[33m%s\\033[0m - \\033[34m%s\\033[0m\\\\n\\\", \\$1, substr(\\$2, 2, length(\\$2)-2)}'\""

# Kullanımı
git au
```

Repository authorlarını, anonim GitHub kullanıcılarını hariç tutarak, sade bir formatta listeler. Projeye katkıda bulunanları görmek için idealdir.

> Bu yazım tarzının neden diğerlerinden biraz farklı olduğunu ve `!sh -c` ile başladığını merak ediyor olabilirsiniz. Bu, komut daha karmaşık shell işlemleri içerdiğinde (örneğin `|`), mantıksal işlemler (`&&`) veya döngüler veya koşullar gibi gelişmiş shell özellikleri gerektirdiğinde kullanılmalıdır.

### 7. fzf ile Lokal Branche Geçiş

```bash
# Alias
clb = "!sh -c 'git branch | grep -v \"^\\*\" | fzf --height=20% --reverse --info=inline | xargs git checkout'"

# Kullanımı
git clb
```

<Callout>
  Bu alias ve aşağıdaki iki aliası kullanabilmek için bilgisayarınızda
  [fzf](https://github.com/junegunn/fzf) kurulu olmalıdır. Kurulum seçenekleri için [bu
  bağlantıya](https://github.com/junegunn/fzf?tab=readme-ov-file#installation) göz atabilirsiniz.
</Callout>

`fzf` kullanarak local branchleri listeler ve etkileşimli olarak checkout yapmanızı sağlar. Çok sayıda branch olan projelerde büyük kolaylık sağlar.

### 8. fzf ile Remote Branche Geçiş

```bash
# Alias
crb = "!sh -c 'git branch -r | grep -v \"^\\*\" | sed \"s|origin/||\" | fzf --height=20% --reverse --info=inline | xargs git checkout'"

# Kullanımı
git crb
```

Yukarıdaki alias ile aynı mantıkta çalışır; tek fark remote branchleri listelemesidir.

### 9. fzf ile Birden fazla Dosyayı Staging’e Ekleme

```bash
# Alias
afz = "!sh -c 'git ls-files -m -o --exclude-standard | fzf -m --print0 | xargs -0 git add'"

# Kullanımı
git afz
```

`fzf` kullanarak dosyaları çoklu seçimle stage etmenizi sağlar. Ok tuşlarıyla gezinip, tab ile seçim yapabilir ve enter ile ekleyebilirsiniz.

### 10. Tüm Git Aliaslarını Listeleme

```bash
# Alias
la = "!sh -c 'git config -l | grep alias | cut -c 7-'"

# Kullanımı
git la
```

Konfigürasyon dosyanıza eklenmiş tüm Git aliaslarını listeler. Aliaslara alışana kadar oldukça kullanışlıdır.

### Sonuç

Git aliasları, Git’in ne kadar esnek ve güçlü olduğunu gösteren harika bir örnektir. Çalışma şeklinize uygun düzenlemeler yaparak verimliliğinizi ciddi şekilde artırabilirsiniz. Commit geri alma, branch yönetimi, authorları görüntüleme ve dosya stage etme gibi günlük işlemleri hızlandıran bu aliaslar, Git kullanım deneyiminizi bir üst seviyeye taşır. Siz de kendi akışınıza uygun başka aliaslar bulabilir ve Git deneyiminizi iyileştirebilirsiniz.
